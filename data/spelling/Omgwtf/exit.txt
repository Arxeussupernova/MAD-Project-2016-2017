<div class="dicTl">Word Origin & History</div>
<br/><span class="me">exit</span><br /><div class="body">1530s, from L. exit "he or she goes out," third person sing. pres. indicative of exire "go out," from ex- "out" + ire "go." Also from L. exitus "a leaving, a going out," noun of action from exire. Originally in English as a stage direction (late 15c.); sense of "door for leaving" is 1786. The verb is </div><a class="more"></a><span class="moreun"><div class="body">c.1600, from the noun; it ought to be left to stage directions and the clunky jargon of police reports. Related: Exited; exiting.</div><a class="less"></a></span>

<p><h3><b>exit definition</b></h3><br />1. <programming> A <a id="dic_result" href="http://dictionary.reference.com/browse/library function">library function</a> in the <a id="dic_result" href="http://dictionary.reference.com/browse/C">C</a> and <a id="dic_result" href="http://dictionary.reference.com/browse/Unix">Unix</a> <a id="dic_result" href="http://dictionary.reference.com/browse/run-time library">run-time library</a> that causes the program to terminate and return control to the <a id="dic_result" href="http://dictionary.reference.com/browse/shell">shell</a>. The alternative to calling "exit" is simply to "fall off the end" of the program or its top-level, <a id="dic_result" href="http://dictionary.reference.com/browse/main">main</a>, routine. <br />Equivalent functions, possibly with different names, exist in pretty much every programming language, e.g. "exit" in <a id="dic_result" href="http://dictionary.reference.com/browse/Microsoft DOS">Microsoft DOS</a> or "END" in <a id="dic_result" href="http://dictionary.reference.com/browse/BASIC">BASIC</a>. <br />On exit, the <a id="dic_result" href="http://dictionary.reference.com/browse/run-time system">run-time system</a> closes open files and releases other resources. An <a id="dic_result" href="http://dictionary.reference.com/browse/exit status">exit status</a> code (a small integer, with zero meaning OK and other values typically indicating some kind of error) can be passed as the only argument to "exit"; this will be made available to the shell. Some languages allow the programmer to set up exit handler code which will be called before the standard system clean-up actions. <br />2. Any point in a piece of code where control is returned to the caller, possibly activating one or more user-provided exit handlers. This might be a <a id="dic_result" href="http://dictionary.reference.com/browse/return">return</a> statement, exit call (in sense 1 above) or code that raises an error condition (either intentionally or unintentionally). If the exit is from the top-level routine then such a point would typically terminate the whole program, as in sense 1. <br />(2008-05-15) <br /></p>
